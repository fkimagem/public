Tutorial da Biblioteca DisplayFK

Esta biblioteca fornece uma solução completa para gerenciar interfaces gráficas em displays TFT usando ESP32. Com uma variedade de widgets personalizáveis, você pode criar interfaces ricas e funcionais para seus projetos de forma simples.

[toc]

Instalação

  1. Baixe a biblioteca DisplayFK
  2. Coloque a pasta da biblioteca no diretório: [Diretório do Arduino]/libraries/
    • No Windows: Documentos/Arduino/libraries/
    • No macOS: ~/Documents/Arduino/libraries/
    • No Linux: ~/Arduino/libraries/
  3. Reinicie o Arduino IDE se estiver aberto

Configuração Inicial

Arquivo user_setup.h

O arquivo user_setup.h permite configurar diversos parâmetros da biblioteca para atender às suas necessidades específicas. Você pode definir os pinos a serem usados para o display e touch, assim como outras configurações importantes.

Principais definições:

//
#define DISP_DEFAULT
// Se possui touch e mod
#define HAS_TOUCH
#define TOUCH_XPT2046
// Pinos
#define DISP_CS 10 //CS, SS, nSS
#define DISP_MISO 13 //MISO, SDO, SOMI
#define DISP_MOSI 11 //MOSI, SDI, SIMO
#define DISP_SCLK 12 //SmodeloCLK, SCK, CLK
#define DISP_RST 9 //RST, RES, RESET
#define DISP_DC 8 //DC, RS, A0
#define TC_CS 21 //CS, SS, nSS
// Frequencia de comunicação
#define DISP_FREQUENCY 27000000
#define TOUCH_FREQUENCY 2500000

Modifique estes valores para corresponder à sua configuração de hardware.

Dicas e Boas Práticas

  1. Inicialização Correta: Sempre certifique-se de inicializar a biblioteca DisplayFK corretamente, chamando myDisplay.createTask() para começar a processar os eventos de toque.
  2. Funções de Callback: Lembre-se de declarar os protótipos das funções de callback no início do código.
  3. Armazenamento de Valores: Guarde os valores dos widgets em variáveis globais e atualize-as nas funções de callback.
  4. Atualização da Tela: Use myDisplay.drawWidgetsOnScreen(screen_number) para desenhar todos os widgets associados a uma tela específica.
  5. Múltiplas Telas: Crie funções separadas para cada tela (screen0(), screen1(), etc.) e mude entre elas usando WidgetBase::loadScreen = screenX;.
  6. Economia de Memória: Se não estiver usando algum tipo de widget, você pode comentar sua inicialização para economizar memória.

Solução de Problemas

  • Display Não Responde: Verifique as conexões de hardware e certifique-se de que os pinos definidos em user_setup.h correspondem à sua configuração física.
  • Touch Não Funciona: Certifique-se de que a calibração do touch está correta. Use myDisplay.recalibrate() para recalibrar se necessário.
  • Widgets Não Aparecem: Verifique se você está chamando myDisplay.setXXX() corretamente para cada tipo de widget e se o número da tela (último parâmetro na criação do widget) corresponde à tela que você está tentando exibir.
  • Callbacks Não São Chamadas: Certifique-se de que a função de callback foi passada corretamente na configuração do widget e que a função está declarada no escopo global.
