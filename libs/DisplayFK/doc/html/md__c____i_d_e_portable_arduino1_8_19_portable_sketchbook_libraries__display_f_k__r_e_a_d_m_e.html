DisplayFK - Framework para Interface Gráfica em Arduino

Visão Geral

O DisplayFK é um framework C++ desenvolvido para facilitar a criação de interfaces gráficas em microcontroladores baseado na biblioteca Arduino_GFX. Este projeto oferece uma coleção abrangente de widgets e componentes de interface que podem ser facilmente integrados em projetos Arduino.

Características Principais

Widgets Suportados

O framework suporta uma variedade de widgets para criar interfaces interativas:

Widgets de Controle

  • TouchArea: Áreas sensíveis ao toque
    • Permite criar áreas interativas na tela
    • Suporta eventos de toque e gestos
    • Configuração: setTouchArea(TouchArea *array, uint8_t amount)
    • Método setup: setup(uint16_t _width, uint16_t _height, functionCB_t &_cb)
      • _width: Largura da área sensível ao toque em pixels
      • _height: Altura da área sensível ao toque em pixels
      • _cb: Função de callback que será chamada quando a área for tocada
  • CheckBox: Caixas de seleção
    • Permite seleção múltipla de opções
    • Estado visual de seleção
    • Configuração: setCheckbox(CheckBox *array, uint8_t amount)
    • Método setup: setup(uint16_t _width, uint16_t _height, uint16_t _color, functionCB_t &_cb)
      • _width: Largura da caixa de seleção em pixels
      • _height: Altura da caixa de seleção em pixels
      • _color: Cor da caixa de seleção quando marcada
      • _cb: Função de callback que será chamada quando o estado mudar
  • CircleButton: Botões circulares
    • Botões com formato circular
    • Suporte a ícones e texto
    • Configuração: setCircleButton(CircleButton *array, uint8_t amount)
    • Método setup: setup(uint16_t _radius, uint16_t _color, functionCB_t &_cb)
      • _radius: Raio do botão circular em pixels
      • _color: Cor do botão quando pressionado
      • _cb: Função de callback que será chamada quando o botão for pressionado
  • RectButton: Botões retangulares
    • Botões com formato retangular
    • Personalização de cores e bordas
    • Configuração: setRectButton(RectButton *array, uint8_t amount)
    • Método setup: setup(uint16_t _width, uint16_t _height, uint16_t _pressedColor, functionCB_t &_cb)
      • _width: Largura do botão em pixels
      • _height: Altura do botão em pixels
      • _pressedColor: Cor do botão quando pressionado
      • _cb: Função de callback que será chamada quando o botão for pressionado
  • TextButton: Botões com texto
    • Botões com foco em texto
    • Suporte a diferentes fontes e tamanhos
    • Configuração: setTextButton(TextButton *array, uint8_t amount)
    • Método setup: setup(const char* _text, const GFXfont* _font, uint16_t _color, functionCB_t &_cb)
      • _text: Texto a ser exibido no botão
      • _font: Ponteiro para a fonte a ser utilizada
      • _color: Cor do texto
      • _cb: Função de callback que será chamada quando o botão for pressionado
  • Toggle: Interruptores
    • Alterna entre dois estados
    • Feedback visual do estado atual
    • Configuração: setToggle(ToggleButton *array, uint8_t amount)
    • Método setup: setup(uint16_t _width, uint16_t _height, uint16_t _color, functionCB_t &_cb)
      • _width: Largura do interruptor em pixels
      • _height: Altura do interruptor em pixels
      • _color: Cor do interruptor quando ativado
      • _cb: Função de callback que será chamada quando o estado mudar
  • RadioGroup: Grupos de botões de rádio
    • Seleção única entre múltiplas opções
    • Gerenciamento automático de estados
    • Configuração: setRadioGroup(RadioGroup *array, uint8_t amount)
    • Método setup: setup(uint16_t _width, uint16_t _height, uint16_t _color, functionCB_t &_cb)
      • _width: Largura do botão de rádio em pixels
      • _height: Altura do botão de rádio em pixels
      • _color: Cor do botão quando selecionado
      • _cb: Função de callback que será chamada quando a seleção mudar
  • HSlider: Barras deslizantes horizontais
    • Controle deslizante horizontal
    • Valores numéricos com range configurável
    • Configuração: setHSlider(HSlider *array, uint8_t amount)
    • Método setup: setup(uint16_t _width, uint16_t _height, uint16_t _color, functionCB_t &_cb)
      • _width: Largura total da barra deslizante em pixels
      • _height: Altura da barra deslizante em pixels
      • _color: Cor da barra deslizante
      • _cb: Função de callback que será chamada quando o valor mudar
  • SpinBox: Campos numéricos com incremento/decremento
    • Entrada numérica com controles
    • Validação de range e incrementos
    • Configuração: setSpinbox(SpinBox *array, uint8_t amount)
    • Método setup: setup(uint16_t _width, uint16_t _height, uint16_t _step, int _min, int _max, uint16_t _color, functionCB_t &_cb)
      • _width: Largura do campo em pixels
      • _height: Altura do campo em pixels
      • _step: Valor de incremento/decremento
      • _min: Valor mínimo permitido
      • _max: Valor máximo permitido
      • _color: Cor do texto
      • _cb: Função de callback que será chamada quando o valor mudar

Recursos Adicionais

  • Suporte a teclado virtual para entrada de texto
  • Teclado numérico para entrada de números
  • Gerenciamento de arquivos SD
  • Sistema de logging
  • Suporte a imagens PNG
  • Calibração de tela touch
  • Gerenciamento de transações de tela
  • Watchdog Timer configurável

Arquitetura

Sistema de Widgets

  • Cada widget é gerenciado individualmente
  • Sistema de configuração flexível para cada tipo de widget
  • Suporte a múltiplas instâncias de cada tipo de widget
  • Gerenciamento de estado e atualização automática

Gerenciamento de Memória

  • Alocação dinâmica de recursos
  • Gerenciamento eficiente de memória
  • Suporte a diferentes tipos de displays

Sistema de Logging

  • Buffer circular para mensagens de log
  • Suporte a armazenamento em SD
  • Rotação automática de arquivos de log
  • Formato estruturado de mensagens

Requisitos

  • Arduino IDE
  • Biblioteca Arduino_GFX
  • ESP32 (ou compatível)
  • Display compatível com Arduino_GFX
  • Opcional: SD Card para logging

Uso

  1. Inclua as bibliotecas necessárias
  2. Configure os widgets desejados
  3. Inicialize o framework
  4. Configure os widgets específicos
  5. Implemente a lógica de interação

Exemplo Básico

#include "displayfk.h"
DisplayFK display;
void setup() {
// Inicialização do display
display.startTransaction();
// Configuração dos widgets
Label labels[2];
display.setLabel(labels, 2);
// Configuração de cada label
labels[0].setup("Título", &FreeSans9pt7b, TL_DATUM, TFT_WHITE, TFT_BLACK);
labels[1].setup("Subtítulo", &FreeSans12pt7b, TL_DATUM, TFT_WHITE, TFT_BLACK);
Button buttons[3];
display.setRectButton(buttons, 3);
// Configuração de cada botão
buttons[0].setup(100, 40, TFT_BLUE, buttonCallback);
buttons[1].setup(100, 40, TFT_GREEN, buttonCallback);
buttons[2].setup(100, 40, TFT_RED, buttonCallback);
// Configuração do teclado virtual (se necessário)
display.setupKeyboard();
// Configuração do teclado numérico (se necessário)
display.setupNumpad();
display.finishTransaction();
}
void loop() {
// Lógica principal
}
